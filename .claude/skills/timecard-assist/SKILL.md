# timecard-assist — 作業時間分析・タイムシート支援

claude-timecard MCPサーバーを使って、Claude Codeのセッションログから作業時間を分析し、タイムシート作成を支援するスキル。

---

## トリガー

以下のような発言で発動:
- 「今月の作業時間は？」「何時間働いた？」
- 「タイムシートを作って」「月末処理」
- 「先月と比較して」「ブランチ別の内訳」
- 「週報を書いて」「レポートを生成」

---

## ワークフロー

### Step 1: ヒアリング（必ず最初にやる）

以下を**対話で確認**してから分析に入る。推測で進めない。

#### 必須の確認事項

1. **プロジェクト名**
   - 「どのプロジェクトの時間を見ますか？」
   - `-p` フィルタに使う。部分一致で効く
   - 複数プロジェクトがある場合は1つずつ処理する

2. **対象期間**
   - 「今月分（3/1〜3/31）ですか？先月分ですか？」
   - フリーランスの請求は月単位が多い。デフォルト7日は短すぎる

3. **過去の手動記録の有無**
   - 「過去に手動で記録したタイムシートや作業ログはありますか？あればフォーマットを揃えられます」
   - 過去データがあれば:
     - 出力フォーマットをそれに揃える
     - キャリブレーション（補正係数）の計算に使える
     - 日付、時間、内容の列構成を合わせる

4. **出力フォーマット**
   - 過去データがあればそれに合わせる
   - なければ「どういう形式で欲しいですか？」と聞く
   - 典型的なフォーマット例:
     ```
     日付    時間    作業内容
     3/01    8.5    〇〇の実装とテスト。PRレビュー対応。
     3/02    6.0    バグ修正。△△のリファクタリング。
     ```

5. **勤務状況の確認**
   - 「体調不良で休んだ日や、稼働が少なかった週はありますか？」
   - 「休憩時間は何時間ですか？（1時間？）」→ `--break-start`/`--break-end` に渡す
   - 「リモートの週と出社の週はありますか？」→ 補正係数の location に影響

### Step 2: データ取得

以下のMCPツールを**並列で**呼ぶ:

```
timecard_daily    → 日別Active時間 + ブロック詳細
timecard_tasks    → タスク推定（ブランチベース）
timecard_keywords → キーワード上位（ノイズチェック用）
timecard_streams  → ブランチ別内訳（重複除去）
```

### Step 3: ノイズ検出・クリーニング

キーワード結果を確認し、明らかなノイズがあれば提案:

- `ultrathink` → Claude Codeの指示語。ストップワード候補
- ユーザー名が出ていたら → 自動除外されているか確認
- 業務ドメイン用語の表記揺れ → 同義語登録を提案

```bash
# ノイズ除去
python timecard.py --add-stop ultrathink

# 同義語登録
python timecard.py --add-synonym jump ジャンプ
```

### Step 3.5: AIレポート生成

ノイズ除去後のクリーンなデータでAIレポートを生成する。

```
timecard_report   → Haiku並列→Opus統合のMarkdownレポート生成
```

レポートには各セグメントの具体的な作業内容（何を実装したか、どんなバグを直したか、PRの内容）がAI分析で書かれている。このレポートがStep 4のタイムシート作成の根拠になる。

### Step 4: タイムシート作成

**`timecard_daily_report`** で日付ごとのAIレポートセグメントを取得し、タイムシートの「作業内容」欄を書く。

#### 作業内容の記述ルール

- **ブランチ名やキーワードから推測しない**。AIレポートのセグメント内容を読んで書く
- レポートにはPR番号、具体的なバグ内容、実装の詳細が含まれている
- 1日に複数タスク/セグメントがあれば、重要度順に列挙する
- 「コーディング」「開発」のような曖昧な表現は避ける

#### 出力に含めるべき情報

- **日別**: 日付、Active時間、主な作業内容（AIレポートから）
- **月間サマリー**: 合計時間、稼働日数、日平均
- **タスク別内訳**: 何にどれだけ時間を使ったか（ブランチ単位）

### Step 5: 確認・調整

- 生成したタイムシートをユーザーに見せて確認を取る
- 「この日の内容、合ってますか？」
- 記憶にある作業と照合してもらう
- 修正があれば反映

## 使えるMCPツール一覧

| ツール | 用途 | 主なパラメータ |
|--------|------|---------------|
| `timecard_daily` | 日別Active時間+ブロック詳細 | start, end, project |
| `timecard_detail` | 特定日の詳細ブロック情報 | start, end, project, date |
| `timecard_streams` | ブランチ別内訳（union-of-intervals） | start, end, project |
| `timecard_tasks` | タスク自動推定 | start, end, project, mode |
| `timecard_keywords` | キーワードTF-IDFランキング | start, end, project, top_n |
| `timecard_compare_months` | 月間比較 | month1, month2, project |
| `timecard_report` | AIレポート生成（Haiku→Opus） | start, end, project, model, depth |
| `timecard_daily_report` | 日付×AIレポートセグメントのマッピング | start, end, project |
| `timecard_graph` | グラフPNG生成 | start, end, project |

## カスタマイズコマンド

ユーザーとの対話中にキーワード精度を改善する:

```bash
# ストップワード追加（ノイズ語の除外）
python timecard.py --add-stop <word1> [word2 ...]

# 同義語登録（表記揺れの統合）
python timecard.py --add-synonym <canonical> <alias1> [alias2 ...]

# Sudachi同義語辞書の一括インポート
python timecard.py --import-sudachi <path-or-url>
```

---

## 注意事項

- Active時間はメッセージ間隔から推定した「正味作業時間」。実際の請求時間とはズレがある
- 過去の手動記録があれば補正係数を計算できる（`--calibration`）
- キーワードにはプロジェクト固有のドメイン用語が出る。ユーザーに内容の正確性を必ず確認する
- 出力結果にはユーザーの業務情報が含まれる。外部共有時は注意
